================================================================================
  CECB PLUS — Assistant CECB (Architecture projet-centree)
================================================================================

OBJECTIF
--------
Application web projet-centree pour l'assistance a la redaction de rapports CECB
(Certificat Energetique Cantonal des Batiments). L'application regroupe quatre
outils complementaires dans un espace de travail unifie par projet :
texte CECB, annexe photos, variantes d'assainissement, et textes CECB (ancienne version).

ARCHITECTURE
------------
Deux pages principales :
  accueil.html — Creation/gestion de projets par adresse (recherche Swisstopo)
  projet.html  — Espace de travail par projet avec 4 onglets

Flux utilisateur :
  1. Connexion (auth.js) — option "Rester connecte" (localStorage persistent)
  2. Accueil : rechercher une adresse -> creer un projet
  3. Projet : naviguer entre les 4 onglets (Texte CECB, Photos, Variantes, Textes)
  4. Donnees sauvegardees automatiquement par projet

PAGES
-----
1. Accueil (accueil.html)
   - Barre auth avec gestion utilisateurs
   - Recherche adresse via API Swisstopo (SearchServer, Height, RegBL find)
   - Auto-remplissage depuis RegBL : EGID, annee, commune, canton, type SIA,
     etages (gastw), logements (ganzwhg), altitude
   - Types SIA : 1020 (individuel), 1025 (collectif), 1030-1120 (autres)
   - Stockage des coordonnees LV95 (easting/northing) depuis Swisstopo
   - Creation de projet avec toutes les metadonnees adresse
   - Liste des projets par utilisateur avec options ouvrir/supprimer
   - Parametres API Claude (cle + modele) — visible uniquement pour les admins

2. Espace projet (projet.html)
   - Barre navigation avec lien retour accueil
   - Barre info projet (nom, canton)
   - 4 onglets avec initialisation paresseuse (lazy init)

   Onglet 1 : Texte CECB (recueil.js)
     - 3 sous-onglets : Enveloppe / Technique / Subventions PB
     - Sous-onglet Enveloppe :
       A. Donnees du batiment (adresse, canton, commune, EGID, annee,
          type SIA, SRE, altitude, etages, logements, coordonnees E/N LV95)
       Donnees RegBL — Logements (visible uniquement pour SIA 1020/1025,
          fetch async depuis Swisstopo RegBL API, affiche ganzwhg + wazim1-6)
       B. Toit (config, type, annee, isolation avec epaisseur+materiau, etat, PV)
       C. Murs contre exterieur (composition, revetement, annee, isolation, etat)
       C2. Murs contre terre (type, isolation, etat)
       C3. Murs c/ non chauffe (type, isolation, etat)
       D. Fenetres (type, vitrage, annee, etat, store)
       E. Sols c/ terre (config, isolation, etat)
       E2. Sols c/ non chauffe (config, isolation, etat)
     - Sous-onglet Technique :
       F. Ventilation (type, recup chaleur)
       G. Chauffage (source, distribution, regulation, appoint, conso)
       H. Eau chaude sanitaire (source, type, solaire)
       I. Appareils electriques (electromenager, eclairage)
       J. Photovoltaique (existant, puissance, batterie)
     - Sous-onglet Subventions PB :
       Lien vers francsenergie.ch/fr
       Recapitulatif des subventions Programme Batiments (calcul auto)
     - Bouton "Importer transcript Plaud" (toggle panel)
     - Generation de texte par section : bouton "Generer le texte" par section
     - Sortie inline avec deux textareas : Etat Initial + Ameliorations
     - Compteur de caracteres par textarea
     - Bouton "Copier" + "Enhance" (API Claude) + "Annuler" par textarea
     - Bouton "Sauvegarder le projet" sur chaque sous-onglet
     - Moteur BLOCS avec templates et placeholders {year}, {isolation_desc}, etc.
     - 12 generateurs de section : generateToitText, generateMursExtText,
       generateMursTerreText, generateMursNcText, generateFenetresText,
       generateSolsTerreText, generateSolsNcText, generateVentilationText,
       generateChauffageText, generateEcsText, generateAppareilsText, generatePvText
     - enhanceField() : reecriture factuelle (Claude API), supprime les marqueurs
       [a completer], bouton Annuler pour revenir au texte initial
     - enrichField() : enrichissement IA avec instruction custom (Claude API)
     - Export PDF du recueil

   Onglet 2 : Annexe Photos (photos.js)
     - Import de photos par glisser-deposer ou selection de fichiers
     - Reorganisation des photos par glisser-deposer
     - Ajout de legendes sous chaque photo
     - Suppression individuelle de photos
     - Generation d'un PDF A4 (2 colonnes x 3 lignes)
     - Nom du projet automatique depuis ProjectStore
     - Bouton "Sauvegarder le projet"

   Onglet 3 : Variantes (variantes.js)
     - Matrice de selection pour 3 variantes d'assainissement energetique
     - Textes-cadre editables (introduction, conclusions, description)
     - Bibliotheque de textes pour chaque poste (toiture, facade, fenetres, etc.)
     - Apercu en temps reel par variante et rapport complet
     - Tableau recapitulatif exportable (copie Word, PNG, HTML)

   Onglet 4 : Textes (ancienne version) (textes.js)
     - Charge des bibliotheques JSON par categorie CECB
     - Onglets groupes : "Enveloppe du batiment" et "Technique du batiment"
     - Pour chaque categorie : formulaire parametres avec sections EI et AM
     - Assemblage automatique de texte via templates JSON avec placeholders
     - Bouton copier + editeur libre

MODELE DE DONNEES
-----------------
localStorage key: "cecb-projects"
Chaque projet contient :
  - id, name, userId, created, updated
  - address: { label, egid, year, altitude, commune, canton, type, floors,
    dwellings, coords: {easting, northing} }
  - recueil: { formData, generatedTexts, transcript }
  - variantes: { variantNames, variantCosts, checks, selectedTexts, customTexts, frameworkTexts }
  - textes: { valeurs, textesGeneres }

IndexedDB: "cecb-photos-db" / store "photos"
  - Cle: projectId -> tableau de photos [{src, caption}]

PERSISTANCE
-----------
- Auto-save debounce (2-3s) sur chaque onglet via ProjectStore
- Photos en IndexedDB (trop volumineuses pour localStorage)
- Cle API Anthropic en localStorage separement (pas par projet)
- Bibliotheques JSON chargees a la demande
- Textes generes sauvegardes par ID de textarea (gen-toit-ei, gen-toit-ap, etc.)
- Session auth : sessionStorage (defaut) ou localStorage ("Rester connecte")

- Option "Rester connecte" : session persistante via localStorage

--------------------------------------------
Dossier bibliotheque/ contenant les fichiers JSON par categorie :
  batiment.json, tcext.json, mcext.json, fen.json, asols.json,
  chauffage.json, ecs.json, elec.json, photo.json, variantes.json

Chaque fichier JSON contient :

LOGIQUE DE GENERATION
---------------------
- Annee par defaut : chaque section enveloppe utilise l'annee de renovation du
  composant si renseignee, sinon l'annee de construction du batiment (meta-year)
  provenant du RegBL ou saisie manuellement.
  Ex: toitYear = rv('toit-year') || rv('meta-year')
  Ex: fenYear = rv('fen-year') || rv('meta-year')
  Ex: mursYear = rv('murs-year') || rv('meta-year')
- Enhance (enhanceField) : correction grammaticale/orthographique + fluidite legere
  Prompt : "Corrige uniquement grammaire, orthographe, ponctuation. Ameliore legerement
  la fluidite mais ne reformule PAS et n'ajoute AUCUN mot/expression.
  Interdit : 'nous constatons', 'nous observons', 'il est a noter', etc.
  Supprime les passages [a completer]. Texte doit rester proche de l'original."
  Apres enhance : bouton "Annuler" pour restaurer le texte initial.

API EXTERNES
------------
- Swisstopo SearchServer : recherche d'adresses suisses
  URL: api3.geo.admin.ch/rest/services/api/SearchServer
- Swisstopo Height API : altitude a partir de coordonnees
  URL: api3.geo.admin.ch/rest/services/height
- Swisstopo RegBL (find) : donnees batiment (EGID, annee, type, etages, logements)
  URL: api3.geo.admin.ch/rest/services/api/MapServer/find
  Parametres: layer=ch.bfs.gebaeude_wohnungs_register&searchField=strname_deinr
  Filtrage par NPA (dplz4) et nom de rue exact
  Champs: egid, gbauj, gastw, ganzwhg, ggdename, gdekt, gkat
  Note: attrs.y = easting (E), attrs.x = northing (N) dans SearchServer
- Anthropic Claude API : reecriture de textes et analyse de transcripts
  Header requis: anthropic-dangerous-direct-browser-access: true
